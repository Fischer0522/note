# 码出高效
## 面向对象
### OOP

OOP三大特性：封装、继承、多态、(抽象，三大天王总有四个)

- 对于封装，在设计模式中的七大原则之一迪米特法则即为A模块使用B模块中的一个方法，要对其除此行为以外的信息知道的尽可能少，只关注对外表达，不关注内部实现，在封装中的体现便是通过private来控制访问权限，模块A与模块B之间仅有的联系为方法和方法的参数和返回值

- 对于继承，应当注意的为是否符合里氏替换法则，即能出现父类的地方，总能够使用子类来替换，如果父类的功能子类无法去实现，则造成了方法污染，除此之外还应当避免方法爆炸，即多次继承之后，继承树上的方法达到上百个之多

- 多态

  - overload：重载，方法名称相同但是参数不同，为一个类内中名称相同但是针对有所不同的一组方法，可以视为针对同一个问题的不同情况的解决方法，处于一种平行关系
  - override：覆写，指在实现接口时，或者继承时，保持方法签名完全相同，子类对于父类特性的一种全新，多样的表达，为一种垂直关系

  多态更应当指对于覆写这类特性，作为继承的一种特性进化，子类继承于父类，而对于父类的特性又有着新的表达

### 类

**抽象类与接口**

抽象类为模板式设计 ，接口为契约时设计 

抽象类是作为一个模板，提供一套规范，任何想要继承这个抽象类的都必须遵守这一套规范，使用这一套模板，比如不同品牌的汽车可以在汽车这个抽象概念的基本共同特征，底盘、轮胎、发动机等

接口为一个契约，定义了方法名参数，返回值等，任何想要实现这个接口的类就必须能够实现这些功能,对于车辆，必须具有刹车这个功能，但是刹车的具体实现，比如刹车方式，刹车时间等由具体的车辆覆写后实现

- 抽象类遵从 is-a关系：Eagle is a bird ，bird is Object 
- 接口遵从 can-do关系：Plane can fly, bird can fly
